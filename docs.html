Drone Control Web UI

A standalone multi-page web app that connects to a MAVLink drone over WebSocket. Open the HTML files directly in any browser — no build step, no framework, no server needed to host the frontend.

ℹ️ This site is purely a frontend. It talks to a Python Flask+SocketIO server running on your Raspberry Pi via HTTP/WebSocket. You must implement and run that server for anything to work.

What each page does

  • Dashboard (index.html) — live telemetry grid (20 fields including rangefinder), ARM/DISARM/TAKEOFF/LAND/RTL/HOLD, emergency stop, flight mode selector, console log.
  • Controller (control.html) — gamepad-style directional controls (D-pad), altitude up/down, yaw rotate, hold-to-repeat movement, keyboard support (WASD/arrows/Q/E/R/F), haptic feedback on mobile.
  • Settings (settings.html) — safety switch, level calibration, MAVLink parameter read/write, server health check, live camera feed, command queue management, WiFi network scan & connect.
  • Docs (docs.html) — this page.

Architecture

Two independent pieces that connect over your local network or via ngrok tunnel:

┌─────────────────────────────────┐ ┌──────────────────────────────────────┐ │ Browser (Phone / PC) │ │ Raspberry Pi 2 │ │ │ │ │ │ index.html / control.html ◄───┼─────┼─► Flask + Flask-SocketIO │ │ settings.html / docs.html │ │ └─ eventlet async worker │ │ │ │ │ │ static/js/core.js │ │ new_api_server.py │ │ ├─ socket.io client │ WS │ ├─ emits 'telemetry' at 5 Hz │ │ │ telemetry events ◄───────┼─────┼────┘ │ │ └─ fetch() REST calls ───────┼─────┼──► POST /arm, /takeoff, /move … │ │ │ │ │ │ localStorage: droneUrl │ │ DroneKit │ └─────────────────────────────────┘ │ └─ /dev/ttyAMA0 @ 57600 baud │ │ MAVLink ──► Pixhawk FC │ └──────────────────────────────────────┘

The browser stores the server URL once in localStorage and reuses it across all pages. All communication uses plain HTTP (or HTTPS via ngrok). The WebSocket connection streams telemetry; REST calls send commands.

Quick Start

1
Install Python dependencies on the Pi
pip install flask flask-socketio eventlet dronekit pymavlink
2
Run the server on the Pi
python new_api_server.py — starts on port 5000 by default.
3
Open the web app
Desktop: open index.html directly in Chrome/Firefox.
Mobile: serve from the Pi — cd webapp && python -m http.server 8080 then visit http://<pi-ip>:8080.
4
Connect
Enter the server URL in the banner at the top — e.g. http://192.168.1.42:5000 or your ngrok URL. The URL saves to localStorage and is reused on every page.
5
Remote access (optional)
ngrok http 5000 on the Pi gives you an HTTPS URL accessible from anywhere. Paste that into the URL banner. Kill ngrok when done — the URL dies with it.

Server Setup — Minimal Template

Below is the minimum Flask-SocketIO server skeleton that this web app expects. Expand it with your DroneKit logic.

new_api_server.py — skeleton 
import eventlet
eventlet.monkey_patch()

from flask import Flask, request, jsonify
from flask_socketio import SocketIO
import dronekit, threading, time

app = Flask(__name__)
app.config['SECRET_KEY'] = 'drone-secret'

socketio = SocketIO(
    app,
    cors_allowed_origins='*',   # allow any origin (browser opens file://)
    async_mode='eventlet',
    ping_timeout=20,
    ping_interval=10,
)

# ── Connect to vehicle ──────────────────────────────────
vehicle = dronekit.connect(
    '/dev/ttyAMA0',
    baud=57600,
    wait_ready=True,
    heartbeat_timeout=30,
)

# ── Telemetry broadcast (5 Hz) ─────────────────────────
def tel_loop():
    while True:
        socketio.emit('telemetry', get_status_data())
        time.sleep(0.2)

threading.Thread(target=tel_loop, daemon=True).start()

# ── Helper: build telemetry dict ───────────────────────
def get_status_data():
    try:
        vibe = vehicle.vibration
        vx = round(vibe.vibration_x, 3) if vibe else None
        vy = round(vibe.vibration_y, 3) if vibe else None
        vz = round(vibe.vibration_z, 3) if vibe else None
    except Exception:
        vx = vy = vz = None

    loc = vehicle.location.global_relative_frame
    att = vehicle.attitude
    bat = vehicle.battery
    return {
        'mode':          str(vehicle.mode.name),
        'armed':         vehicle.armed,
        'altitude':      round(loc.alt, 3) if loc.alt else 0,
        'lat':           round(loc.lat, 6) if loc.lat else None,
        'lon':           round(loc.lon, 6) if loc.lon else None,
        'groundspeed':   round(vehicle.groundspeed, 2),
        'airspeed':      round(vehicle.airspeed, 2),
        'pitch':         round(att.pitch, 4) if att else None,
        'roll':          round(att.roll, 4) if att else None,
        'yaw':           round(att.yaw, 4) if att else None,
        'battery':       round(bat.voltage, 2) if bat and bat.voltage else None,
        'battery_level': bat.level if bat else None,
        'satellites':    vehicle.gps_0.satellites_visible,
        'gps_fix':       vehicle.gps_0.fix_type,
        'ekf_ok':        vehicle.ekf_ok,
        'is_armable':    vehicle.is_armable,
        'vibe_x': vx, 'vibe_y': vy, 'vibe_z': vz,
    }

# ── CORS headers ───────────────────────────────────────
@app.after_request
def add_cors(r):
    r.headers['Access-Control-Allow-Origin']  = '*'
    r.headers['Access-Control-Allow-Headers'] = 'Content-Type, ngrok-skip-browser-warning'
    r.headers['Access-Control-Allow-Methods'] = 'GET,POST,OPTIONS'
    return r

@app.route('/', defaults={'path':''}, methods=['OPTIONS'])
@app.route('/<path:path>', methods=['OPTIONS'])
def options(_):
    return '', 204

# ── Run ────────────────────────────────────────────────
if __name__ == '__main__':
    socketio.run(app, host='0.0.0.0', port=5000, debug=False)
⚠️ CORS must be open. The browser opens index.html from file:// or a different port, so the server has no shared origin with the Pi. Without cors_allowed_origins='*' in SocketIO and the after_request CORS headers, every request will be blocked.

WebSocket & Telemetry

The site connects with Socket.IO 4.x using WebSocket transport only (no long-polling fallback). Connection options used by core.js: 

io(url, {
  transports: ['websocket'],   // no polling — avoids CORS preflight issues
  upgrade: false,
  reconnectionAttempts: 10,
  reconnectionDelay: 2000,
  timeout: 10000,
})

The server must emit the event named exactly 'telemetry' — this is what socket.on('telemetry', ...) in core.js listens for. Emit it inside your background thread at whatever rate you want; 5 Hz (every 200 ms) works well.

Telemetry Schema

Every field is optional — the UI handles null gracefully (shows --). Missing fields are simply ignored. Add extra fields freely; the UI ignores unknown keys.

armed
bool
True when motors armed
rangefinder

Distance from rangefinder sensor. Used as altitude source when GPS unavailable (GUIDED_NOGPS). Returns 0.0 if sensor absent.
altitude
float (m)
Relative altitude above home from barometer/EKF
lat / lon
float (°)
GPS position, 6 decimal places
groundspeed
float (m/s)
Speed over ground
airspeed
float (m/s)
Airspeed from pitot (if fitted)
pitch / roll / yaw
float (rad)
Attitude angles in radians — UI converts to degrees
battery

API Reference — Flight Commands

All commands return JSON. On success: {"success": true, "message": "..."}. On failure: {"success": false, "error": "reason"}. The UI logs both outcomes in the console card.

MethodEndpointBodyDescription
POST/arm—Arm motors asynchronously (returns immediately). Waits up to 15 s in a daemon thread. Vehicle must pass pre-arm checks (EKF OK, GPS fix or GUIDED_NOGPS, no failures).
POST/disarm—Disarm motors. Blocked if rangefinder > 0.17 m (drone must be on the ground). Returns 400 if altitude check fails.
POST/takeoffaltitude: floatArm + takeoff to altitude in metres (0.5–10 m). Blocked if battery < 13.2 V (4S minimum). With GPS uses simple_takeoff; without GPS uses rangefinder + thrust control to reach target altitude.
POST/land—Set mode to LAND. Vehicle descends and disarms automatically.
POST/rtl—Return to launch. Blocked if GPS fix < 3. Climbs to RTL_ALT, flies home, lands.
POST/hold—Set LOITER mode and cancel any active movement or takeoff. Also cancels hold-to-repeat motion from the controller page.
POST/emergency—Immediate LAND + disarm. Bypasses command lock. Use only on ground or if crash is unavoidable.
POST/modemode: stringSet flight mode by name. Valid: STABILIZE, ALTHOLD, LOITER, GUIDED, GUIDED_NOGPS, LAND, RTL, AUTO, POSHOLD.
Example implementation — /arm (async, non-blocking) 
_cmd_lock = threading.Lock()

def _arm_drone():
    """Daemon thread — releases lock when done."""
    try:
        vehicle.armed = True
        start = time.time()
        while not vehicle.armed:
            if time.time() - start > 15:
                print("Arm timeout")
                return
            time.sleep(0.5)
        print("Armed!")
    finally:
        _cmd_lock.release()

@app.route('/arm', methods=['POST'])
def arm():
    if not _cmd_lock.acquire(blocking=False):
        return jsonify({'success': False, 'message': 'Another command in progress'}), 409
    try:
        if vehicle.armed:
            _cmd_lock.release()
            return jsonify({'success': False, 'message': 'Already armed'})
        t = threading.Thread(target=_arm_drone)
        t.daemon = True
        t.start()
        return jsonify({'success': True, 'message': 'Arming...'})
    except Exception as e:
        _cmd_lock.release()
        return jsonify({'success': False, 'message': str(e)})
Example implementation — /takeoff 
@app.route('/takeoff', methods=['POST'])
def takeoff():
    data = request.get_json(force=True) or {}
    alt  = float(data.get('altitude', 2))
    try:
        vehicle.simple_takeoff(alt)
        return jsonify({'success': True, 'message': f'Taking off to {alt}m'})
    except Exception as e:
        return jsonify({'success': False, 'error': str(e)})

API Reference — Movement

Movement commands are called repeatedly while the pilot holds a direction button (hold-to-repeat at ~350 ms intervals). Each press sends a discrete move command. Release sends /hold.

MethodEndpointBodyDescription
POST /move direction: string
distance: float (m)
speed: float (m/s) 		Move in forward, backward, left, right, up, or down. Uses NED body-frame velocity. Speed must be 0.2–0.3 m/s (indoor AI limit — 400 returned otherwise). Auto-switches to GUIDED_NOGPS if not already in a GUIDED mode. Runs asynchronously; switches back to LOITER when done.
POST /yaw direction: string
degrees: float
speed: int (°/s) 		Rotate left or right by specified degrees. Uses condition_yaw MAVLink command.
Example implementation — /move (NED velocity) 
from pymavlink import mavutil

def send_ned_velocity(vx, vy, vz, duration=0):
    msg = vehicle.message_factory.set_position_target_local_ned_encode(
        0, 0, 0,
        mavutil.mavlink.MAV_FRAME_BODY_NED,
        0b0000111111000111,   # type_mask: enable velocity only
        0, 0, 0,              # position (ignored)
        vx, vy, vz,           # velocity in m/s
        0, 0, 0,              # accel (ignored)
        0, 0                  # yaw, yaw_rate (ignored)
    )
    vehicle.send_mavlink(msg)

@app.route('/move', methods=['POST'])
def move():
    data = request.get_json(force=True) or {}
    direction = data.get('direction', 'forward')
    speed     = float(data.get('speed', 1))

    mapping = {
        'forward':  ( speed, 0,      0),
        'backward': (-speed, 0,      0),
        'right':    (0,       speed, 0),
        'left':     (0,      -speed, 0),
    }
    vx, vy, vz = mapping.get(direction, (0, 0, 0))
    send_ned_velocity(vx, vy, vz)
    return jsonify({'success': True, 'message': f'Moving {direction}'})
Example implementation — /yaw (condition_yaw) 
@app.route('/yaw', methods=['POST'])
def yaw():
    data      = request.get_json(force=True) or {}
    degrees   = float(data.get('degrees', 10))
    speed     = int(data.get('speed', 30))
    clockwise = 1 if data.get('direction', 'right') == 'right' else -1

    msg = vehicle.message_factory.command_long_encode(
        0, 0,
        mavutil.mavlink.MAV_CMD_CONDITION_YAW,
        0,
        degrees,   # heading change
        speed,     # rotation speed deg/s
        clockwise, # direction: 1=CW, -1=CCW
        1,         # relative (1) vs absolute (0)
        0, 0, 0
    )
    vehicle.send_mavlink(msg)
    return jsonify({'success': True, 'message': f'Yawing {data.get("direction")} {degrees}°'})

API Reference — System & Config

MethodEndpointBody / ParamsDescription
GET/health—Returns server health info: connected, armed state, mode, battery, GPS fix, uptime. Used by Settings page health cards.
POST/safetystate: "on"/"off"Enable (on) or disable (off) the ArduPilot hardware safety switch via MAVLink.
POST/lvlcal—Trigger accelerometer level calibration (MAV_CMD_PREFLIGHT_CALIBRATION).
GET/param?name=XQuery param: nameRead a MAVLink parameter by name, e.g. /param?name=WPNAV_SPEED.
POST/paramparam: string value: anyWrite a MAVLink parameter. Value is cast to the correct type automatically by DroneKit.
Example implementation — /param read & write 
@app.route('/param', methods=['GET', 'POST'])
def param():
    if request.method == 'GET':
        name = request.args.get('name', '').upper()
        val  = vehicle.parameters.get(name)
        if val is None:
            return jsonify({'success': False, 'error': f'Param {name} not found'})
        return jsonify({'success': True, 'name': name, 'value': val})

    data  = request.get_json(force=True) or {}
    name  = data.get('param', '').upper()
    value = data.get('value')
    try:
        vehicle.parameters[name] = value
        return jsonify({'success': True, 'message': f'{name} set to {value}'})
    except Exception as e:
        return jsonify({'success': False, 'error': str(e)})

API Reference — Camera & Command Queue

MethodEndpointBodyDescription
GET/video_feed—MJPEG stream from Pi camera. Set as <img src="..."> to display live feed. Requires PiCamera or OpenCV capture.
GET/camera/start—Open the V4L2 camera device and start the frame capture loop.
POST/camera/stop—Release the camera and stop the stream.
GET/queue/status—Returns current command queue depth and items. Each item has action and params.
POST/queue/clear—Flush the entire command queue.
POST/queue/addaction: string params: objectPush a command onto the queue (write-only for now — future mission sequencer will auto-execute).
MJPEG video feed — OpenCV + V4L2 (used in new_api_server_1.py) 
import cv2, threading

camera      = None
camera_lock = threading.Lock()
JPEG_Q      = 70   # quality 0–100

def start_camera():
    global camera
    with camera_lock:
        if camera is not None:
            return
    cap = cv2.VideoCapture('/dev/video0', cv2.CAP_V4L2)
    cap.set(cv2.CAP_PROP_FOURCC, cv2.VideoWriter_fourcc(*'MJPG'))
    cap.set(cv2.CAP_PROP_FRAME_WIDTH,  640)
    cap.set(cv2.CAP_PROP_FRAME_HEIGHT, 480)
    cap.set(cv2.CAP_PROP_FPS, 10)
    import time; time.sleep(1)   # let V4L2 driver settle
    with camera_lock:
        if cap.isOpened():
            camera = cap

def gen_frames():
    start_camera()
    while True:
        with camera_lock:
            cap = camera
        if cap is None:
            break
        ok, frame = cap.read()
        if not ok:
            continue
        _, buf = cv2.imencode('.jpg', frame,
                              [cv2.IMWRITE_JPEG_QUALITY, JPEG_Q])
        yield (b'--frame\r\nContent-Type: image/jpeg\r\n\r\n'
               + buf.tobytes() + b'\r\n')

@app.route('/video_feed')
def video_feed():
    return Response(gen_frames(),
                    mimetype='multipart/x-mixed-replace; boundary=frame')

API Reference — WiFi Management

The server can scan and connect to WiFi networks via nmcli on the Pi. Useful for switching the Pi onto a hotspot in the field without SSH access.

MethodEndpointBody / ParamsDescription
GET /wifi/scan
/network/scan		 — Returns list of visible networks. Each entry: ssid, signal (0–100), security. The /network/* paths are aliases used by the Settings page UI.
POST /wifi/connect
/network/connect		 ssid: string password: string Connect Pi to a WiFi network. Uses sudo nmcli device wifi connect. Times out after 30 s if association fails.
Auto-reconnect daemon (running at startup) 
# On startup the server:
# 1. Connects to configured SSID
# 2. Launches wifi_monitor_loop as daemon thread
# 3. Monitor checks connectivity every 15 s and reconnects if dropped

def wifi_monitor_loop(ssid, password=None):
    while True:
        time.sleep(15)
        if not is_wifi_connected():
            scan_networks()
            if is_ssid_available(ssid):
                connect_wifi(ssid, password)

# Startup sequence in __main__:
connect_wifi(ssid=WIFI_SSID, password=WIFI_PASS)
monitor = threading.Thread(target=wifi_monitor_loop, args=(WIFI_SSID, WIFI_PASS))
monitor.daemon = True
monitor.start()

# ngrok tunnel auto-starts after WiFi is up:
ngrok_process = subprocess.Popen(['ngrok', 'http', '5000'], ...)
ℹ️ sudo is required for nmcli device wifi connect. Add the Pi user to sudoers for nmcli, or set NOPASSWD for that command in /etc/sudoers to avoid password prompts.

Hardware Setup

Tested hardware configuration:

  • Flight Controller: Pixhawk 1 / 2.4.8 (ArduCopter firmware)
  • Companion Computer: Raspberry Pi 2 Model B (or newer Pi)
  • Connection: Pixhawk TELEM2 port → Pi UART GPIO (pins 8/10), /dev/ttyAMA0, baud 57600
  • Power: Pi powered from BEC/UBEC, not USB from Pixhawk

Pi UART config

Disable the serial console so the UART is free for MAVLink:

# /boot/config.txt
enable_uart=1
dtoverlay=disable-bt        # Pi 3/4 only — frees ttyAMA0 from Bluetooth

# /boot/cmdline.txt — remove: console=serial0,115200
# Then reboot

Verify the vehicle connects before starting the server:

python3 -c "
import dronekit
v = dronekit.connect('/dev/ttyAMA0', baud=57600, wait_ready=True)
print('Mode:', v.mode.name, '| Armed:', v.armed)
v.close()
"

Remote Access — ngrok

If you host the web app on GitHub Pages or Netlify (HTTPS), the browser will block HTTP calls to the Pi. ngrok tunnels the Pi's HTTP server over HTTPS, solving this.

1
Install ngrok on the Pi: curl -s https://ngrok-agent.s3.amazonaws.com/ngrok.asc | sudo tee /etc/apt/trusted.gpg.d/ngrok.asc && echo "deb https://ngrok-agent.s3.amazonaws.com buster main" | sudo tee /etc/apt/sources.list.d/ngrok.list && sudo apt update && sudo apt install ngrok
2
Authenticate: ngrok config add-authtoken <your-token> (free account at ngrok.com)
3
Start tunnel while server runs: ngrok http 5000 — copy the https://xxxx.ngrok-free.app URL into the web app's URL banner.
4
Kill ngrok when done flying. URL becomes invalid immediately — no residual exposure.
ℹ️ ngrok free tier gives one tunnel, one concurrent connection. Good enough for a single operator. The URL changes every restart unless you configure a static domain (paid plan).
⚠️ ngrok browser warning / CORS. ngrok intercepts requests that lack the ngrok-skip-browser-warning: true header and returns an HTML warning page — which has no CORS headers, causing a CORS block in the browser. The web app sends this header automatically on every request. The server must also allow it in preflight via Access-Control-Allow-Headers: ngrok-skip-browser-warning (the after_request hook in the skeleton above does this).

Serving the Web App

Three options depending on your use case:

MethodHowWorks on mobile?Notes
Open HTML directly Double-click index.html in file explorer No (file:// blocked on Android/iOS) Simplest. Desktop Chrome/Firefox only.
Local HTTP server cd webapp && python -m http.server 8080 on PC or Pi, then visit http://<ip>:8080 Yes (same Wi-Fi) Best option for field use. Pi can serve both the API (port 5000) and the web app (port 8080).
GitHub Pages / Netlify Push webapp/ folder to a repo, enable Pages Yes Requires ngrok on Pi for HTTPS. URL is permanent — no need to re-enter after reconnect.

Recommended field setup: Pi serves the web app on port 8080 and the API on port 5000. Phone connects to Pi's hotspot, opens http://10.42.0.1:8080, enters http://10.42.0.1:5000 as the server URL. No internet needed. 

# On Pi — run both in parallel
python new_api_server.py &
cd /path/to/webapp && python -m http.server 8080
